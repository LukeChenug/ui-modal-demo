先说还 4.7 债的部分

Opus 4.7 是 4 月 16 日发的，4.8 是 5 月 28 日。中间 41 天。 Anthropic 之前没有过这个节奏。

4.7 当时 Reddit 上「Opus 4.7 is not an upgrade but a serious regression」 拿了 2300 赞。X 上说"没比 4.6 进步"的帖子 1.4 万赞。 TechCrunch 直接把 4.8 快速发布归因为 "chilly reception to Opus 4.7"。

  • effort 控制还给用户了（4.7 的 adaptive reasoning 被骂翻）
  • 写代码不那么瞎自信了，会自己标出不确定的地方（官方说降 4 倍幻觉）
  • 价格不变 $5 / $25

41 天发版的节奏逻辑

4.7 和 4.8 的 training data cutoff 都是 2026 年 1 月，完全一样。 base model 没有重做预训练，迭代的是 post-training——RLHF、safety、tool-use 微调。

base model 复用，post-training 快速迭代——
这才是能频繁切片发版的真正原因。

时机刚好赶在 GPT-5.6 传言发布前。 芯片厂商的打法，搬到大模型行业了。

Fast Mode "3 倍便宜"
是给谁的

大部分报道讲错了。"3 倍便宜"是相对自家上一代 Fast Mode， 不是相对标准模式。Fast Mode 还是比标准模式贵 2 倍。

真正的客户是 API 接入的大企业。 微软甚至要求员工停止使用 Claude Code，因为账单肉疼。

对个人开发者来说，订阅档依然最划算。
Fast Mode 降价红包不是发给我们的。

Dynamic Workflows
才是真正的大招

这功能没怎么被讨论，但我觉得是这次最值得关注的。 Claude 可以在运行时自己决定用什么工具、走什么分支。

不是你写好 workflow 它来跑——
是它自己规划 workflow 然后执行。

对设计师来说：以后你描述一个需求，它能自己决定 是先搜资料、再画原型、还是直接出代码。 不需要你手动编排 agent 流程了。

Terminal-Bench 第一
意味着什么

Opus 4.8 在 Terminal-Bench 跑到第一。 这个 benchmark 测的是"在真实终端环境完成复杂任务"。

  • 不是做选择题，是真的开 shell 写代码、调试、部署
  • 和 SWE-bench 不同，它测的是端到端多步骤能力
  • 对 Claude Code 用户来说：体感会明显变好

但也别太迷信。Benchmark 和真实使用之间永远有 gap。 4.7 的 SWE-bench 也很高，但体感被骂惨。

作为设计师
我关注什么

模型迭代到这个阶段，差异化不在"智商"了。 我日常用 AI 辅助设计，真正影响体验的是：

  • 指令跟随的稳定性——同一个 prompt 跑 5 次结果一致吗
  • 工具调用的准确率——MCP 接 Figma 时不瞎调参数吗
  • 上下文窗口的利用率——200K 不是让你复读前文的
  • 拒绝幻觉——不确定就说不确定，别编

4.8 在"降幻觉"这点上有进步。
对设计 agent 来说，这比跑分重要 10 倍。
