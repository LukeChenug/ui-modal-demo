Claude Opus 4.8 发了，聊聊这次更新

2026.05 · 随手聊 · 设计师视角

一、41天出新版，明显是来补锅的

4.7 是 4 月 16 号发的，4.8 是 5 月 28 号，中间就 41 天。Anthropic 之前没有过这个节奏。

为什么这么急？因为 4.7 被骂惨了。Reddit 上有条帖子标题直接写「Opus 4.7 is not an upgrade but a serious regression」，拿了 2300 赞。X 上类似的吐槽也有 1.4 万赞。TechCrunch 后来写 4.8 的时候直接说是因为 "chilly reception to Opus 4.7"。

看了一下，4.7 和 4.8 的 training data cutoff 都是 2026 年 1 月，一模一样。base model 没重训，改的是 post-training 那层——RLHF、safety、tool-use 微调。所以才能这么快出版，不是重新训了个模型。

补了什么：effort 控制还给用户了（4.7 那个 adaptive reasoning 真的离谱，模型自己决定用多少力气想问题）；写代码不瞎编了，会标出不确定的地方，官方说幻觉降了 4 倍；价格没变 $5/$25。

二、Fast Mode 降价，跟咱关系不大

很多报道说"便宜了3倍"，但仔细看是相对自家上一代 Fast Mode 的价格，不是相对标准模式。Fast Mode 本身还是比标准模式贵。

这个降价主要是给大企业的。听说微软内部让员工停用 Claude Code 了，原因就是账单太离谱。Anthropic 降价就是为了留住这批大客户。

对我们个人来说，$20 Pro 或 $100 Max 的订阅档够用了。每月 API 花费低于 $200 的话，订阅比按量计费划算。

三、Dynamic Workflows，我觉得最值得关注

这个功能媒体报道很少，但我觉得是这次最猛的更新。简单说就是：以前你设计好步骤让 AI 跑，现在你说个目标，它自己拆任务、选工具、决定顺序。

Bun 团队拿它在 11 天内把 75 万行 Zig 代码迁到了 Rust。虽然 13000 个 unsafe block 有争议，但这个量级已经不是"辅助编程"了。

对设计师来说：以后你描述一个需求，它自己决定先搜资料还是先画原型还是直接出代码。不用手动编排流程了，省心很多。

四、日常用下来我在意什么

到这个阶段，模型之间的差异已经不在"智商"了。我日常用 AI 辅助设计，真正影响体验的是：

  • 指令跟随稳定不稳定——同一个 prompt 跑 5 次结果一样吗？不一样我工作流就废了
  • 工具调用准不准——MCP 接 Figma 的时候别瞎传参数，改错了比不改更烦
  • 上下文窗口别浪费——200K 是让你记住项目背景的，不是让你复读前文的
  • 不确定就说不确定——别编一个看起来对的答案骗我

4.8 在"不瞎编"这点上确实好了一些。对我来说这比"更聪明"有用。

总的来说，4.8 不是什么大革命，就是把 4.7 搞坏的东西修回来了，顺便加了 Dynamic Workflows。但 41 天就出修复版，说明他们确实在听反馈。这种态度比跑分让人安心。
