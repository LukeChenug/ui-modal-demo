先说还 4.7 债的部分

Opus 4.7 是 4 月 16 日发的，4.8 是 5 月 28 日，中间只有 41 天。Anthropic 之前从没有过这个发版节奏。

为什么这么急？因为 4.7 翻车了。Reddit 上「Opus 4.7 is not an upgrade but a serious regression」拿了 2300 赞。X 上吐槽帖 1.4 万赞。TechCrunch 直接把 4.8 快速发布归因为"chilly reception to Opus 4.7"。

这次还了什么债：

  • effort 控制还给用户了——4.7 的 adaptive reasoning 被骂翻，用户觉得模型在偷懒
  • 写代码不再瞎自信了，会主动标出不确定的地方（官方称降低 4 倍幻觉率）
  • 价格不变 $5/$25，没趁机涨价
  • 上下文窗口利用率提升，不再无意义复读前文

41 天发版的节奏逻辑

4.7 和 4.8 的 training data cutoff 都是 2026 年 1 月，完全一样。base model 没有重做预训练，迭代的是 post-training——RLHF、safety、tool-use 微调。

base model 复用，post-training 快速迭代——这才是能 41 天切一版的真正原因。不是重新训了一个模型，是在同一个底座上反复调教。

时机也很微妙：刚好赶在 GPT-5.6 传言发布前。芯片厂商那套"抢先发布卡位"的打法，已经搬到大模型行业了。

对用户的启示：以后模型更新会越来越频繁，但不代表每次都是"换代"。大部分是同一个底座的调优版本。别每次都急着迁移。

Fast Mode "3 倍便宜"
到底是给谁的

大部分报道讲错了。"3 倍便宜"是相对自家上一代 Fast Mode 的价格，不是相对标准模式。实际上 Fast Mode 依然比标准模式贵近 2 倍。

真正的目标客户是通过 API 接入的大企业。微软内部甚至要求员工停止使用 Claude Code，因为 API 账单太肉疼了。Anthropic 降价就是为了留住这批大客户。

对个人开发者来说，月度订阅档（$20 Pro / $100 Max）依然是最划算的方案。Fast Mode 降价的红包不是发给我们的，是发给企业采购的。

一个判断标准：如果你每月 API 花费低于 $200，订阅比按量计费划算。超过这个数再考虑 Fast Mode。

Dynamic Workflows
才是这次真正的大招

这个功能媒体报道很少，但我认为是 4.8 最值得关注的更新。Claude 现在可以在运行时自己决定用什么工具、走什么分支、按什么顺序执行。

不是你写好 workflow 它来跑——是它自己规划 workflow 然后执行。从"执行者"变成了"规划者+执行者"。

对设计师来说这意味着什么？以后你描述一个需求，它能自己决定是先搜竞品资料、再画线框、还是直接生成代码。你不需要手动编排每一步 agent 流程了。

这也是为什么 Claude Code 体验进步最明显——底层就是 Dynamic Workflows 在驱动多步骤任务的规划和执行。

Terminal-Bench 第一
意味着什么

Opus 4.8 在 Terminal-Bench 登顶。这个 benchmark 和传统选择题不一样，它测的是"在真实终端环境里完成复杂多步骤任务"的能力。

  • 真的开 shell、写代码、装依赖、调试报错、部署服务
  • 和 SWE-bench 的区别：后者只改代码，Terminal-Bench 测端到端
  • 对 Claude Code / Cursor 用户：体感会明显好于 4.7
  • 对 agent 开发者：模型终于能可靠地串联多个工具了

但也别太迷信 benchmark。4.7 的 SWE-bench 分数也很高，但实际体感被骂惨了。跑分和日常使用之间永远有 gap，真正的验证是你自己用一周后的感受。

作为设计师
我真正关注什么

模型迭代到这个阶段，差异化已经不在"智商"了。我日常用 AI 辅助设计工作，真正影响我体验的是这几点：

  • 指令跟随的稳定性——同一个 prompt 跑 5 次，结果一致吗？不一致我就没法建立工作流
  • 工具调用的准确率——MCP 接 Figma 的时候不瞎传参数吗？一个错误调用比不调用更浪费时间
  • 上下文窗口的利用率——200K token 不是让你复读前文的，是让你记住项目上下文
  • 诚实度——不确定就说不确定，别编一个看起来正确的答案骗我

4.8 在"降幻觉"这点上有实质进步。对于做设计 agent 的人来说，模型的诚实度比跑分重要 10 倍。一个会说"我不确定"的模型，比一个"自信地编"的模型有用得多。

总结：不是革命，是补课

4.8 不是革命，是补课。但补课的速度和方向都对了。

41 天一个版本、Terminal-Bench 登顶、Dynamic Workflows 开放——Anthropic 在用行动回应 4.7 的差评。

作为用户，这种"反馈被听到、然后快速修正"的感觉，比任何 benchmark 数字都让人安心。这才是一个产品团队该有的迭代态度。

预计下次更新 7 月中旬左右。到时候再来拆。
