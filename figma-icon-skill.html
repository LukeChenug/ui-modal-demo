执行流程

工作流程

从一句话描述到可用组件，七个步骤，全程自动化。

📝
Step 01

描述需求

用自然语言告诉 Claude 你需要什么图标，包括语义、尺寸和风格。

"画一个24px的设置齿轮图标，线性风格，2px描边"
🧠
Step 02

意图解析

Claude 理解图标语义，提取尺寸、风格、描边等结构化参数。

size: 24, style: "outline", stroke: 2
🎨
Step 03

SVG 生成

基于设计规范生成标准 SVG 路径数据，确保视觉语义准确。

<path d="M12 2a10 10 0..." />
✅
Step 04

质量校验

自动检查像素对齐、描边一致性和视觉重量平衡，不合格则重新生成。

pixel_align: ✓ stroke_weight: ✓ balance: ✓
🔌
Step 05

MCP 调用

通过 Figma MCP 协议将 SVG 数据写入画布，创建矢量节点。

figma.createVector(paths, props)
📐
Step 06

自动整理

按命名规范归类到对应 Frame，统一间距，保持文件整洁有序。

Icons/System/settings/outline
🚀
Step 07

同步发布

更新 Figma 组件库，导出多格式资源，推送 SVG 到代码仓库，生成 Design Token。

publish → component_lib export → git push
